## 一、方案选型对比

### 1.1 主流微前端方案矩阵

| 维度           | micro-app                  | qiankun                   | wujie                      | Module Federation         | iframe          |
| -------------- | -------------------------- | ------------------------- | -------------------------- | ------------------------- | --------------- |
| **技术原理**   | WebComponent + 沙箱        | single-spa + import-html  | WebComponent + iframe 沙箱 | Webpack/Vite 模块共享     | 浏览器原生隔离  |
| **接入成本**   | ⭐ 极低（类组件标签）      | ⭐⭐ 中等（生命周期改造） | ⭐ 低                      | ⭐⭐⭐ 高（构建配置）     | ⭐ 极低         |
| **JS 沙箱**    | ✅ Proxy + iframe 双模式   | ✅ Proxy（单实例限制）    | ✅ iframe 天然隔离         | ❌ 无沙箱                 | ✅ 天然隔离     |
| **CSS 隔离**   | ✅ 自动 scoped             | ⚠️ 需手动处理             | ✅ iframe 天然隔离         | ❌ 无隔离                 | ✅ 天然隔离     |
| **通信机制**   | 数据绑定 + PostMessage     | Actions + Props           | Props + PostMessage        | 共享模块直接调用          | PostMessage     |
| **预加载**     | ✅ `microApp.preFetch()`   | ✅ 支持                   | ✅ 支持                    | ✅ 天然支持               | ❌ 不支持       |
| **Vite 兼容**  | ✅ 原生支持（iframe 沙箱） | ⚠️ 需 vite-plugin-qiankun | ✅ 原生支持                | ✅ vite-plugin-federation | ✅              |
| **多实例**     | ✅ 支持                    | ❌ 不支持                 | ✅ 支持                    | ✅ 支持                   | ✅ 支持         |
| **Keep-Alive** | ✅ 原生属性支持            | ❌ 需自实现               | ✅ 原生支持                | ❌ 不适用                 | ⚠️ display:none |
| **维护状态**   | 🟢 活跃（京东开源）        | 🟡 低频（蚂蚁）           | 🟢 活跃（腾讯）            | 🟢 活跃（Webpack）        | —               |
| **包体积**     | ~15KB                      | ~30KB                     | ~20KB                      | 0（构建时）               | 0               |

### 1.2 为什么选择 micro-app

| 项目需求              | micro-app 方案                        | 匹配度  |
| --------------------- | ------------------------------------- | ------- |
| Vue 3 + Vite 8 技术栈 | 原生支持，无需额外插件                | ✅ 完美 |
| 类 HTML 标签式开发    | `<micro-app>` 标签                    | ✅ 完美 |
| 子应用技术栈无关      | 支持 Vue/React/Angular/jQuery         | ✅ 完美 |
| 主题统一同步          | `data` 绑定 + PostMessage + `setData` | ✅ 完美 |
| 子应用路由保活        | `keep-alive` 属性                     | ✅ 完美 |
| 渐进式接入            | 子应用零改造                          | ✅ 完美 |
| iframe 沙箱模式       | Proxy + iframe 双沙箱                 | ✅ 完美 |
| 预加载                | `microApp.preFetch()`                 | ✅ 完美 |

### 1.3 已知限制与缓解方案

| 限制                             | 影响                         | 缓解方案                            |
| -------------------------------- | ---------------------------- | ----------------------------------- |
| `<micro-app>` 标签名需全小写     | ESLint PascalCase 规则冲突   | `eslint-disable` 单行注释           |
| iframe 沙箱性能略低于 Proxy 沙箱 | 首次加载多 ~50-100ms         | 预加载 `preFetch` + keep-alive 缓存 |
| 子应用路由需统一前缀             | 路由命名空间约束             | 规范化 `baseroute` 配置             |
| 跨域需 CORS 配置                 | 开发环境跨端口调试           | Vite proxy / `cors: true`           |
| iframe 沙箱中 `history` 隔离     | 子应用无法直接操作主应用路由 | PostMessage 通知主应用跳转          |

### 2.2 技术选型

| 层面       | 技术                 | 版本         | 说明              |
| ---------- | -------------------- | ------------ | ----------------- |
| 微前端框架 | @micro-zoe/micro-app | ^1.0.0-rc.29 | 京东开源          |
| 主应用框架 | Vue 3 + Vite 8       | 3.5 / 8.x    | 渐进式框架        |

| 主题系统   | @robot-admin/theme   | —            | Dark/Light/System |

### 2.3 核心设计原则

1. **零改造接入** — 子应用不需要暴露 `bootstrap/mount/unmount`，micro-app 自动渲染
2. **类组件开发** — `<micro-app name="app" url="..." />` 像 Vue 组件一样使用
3. **门户工作台** — 统一的系统入口，三栏布局聚合所有业务系统
4. **主题统一** — 主应用 Dark/Light 模式实时同步到所有子应用
5. **渐进式演进** — 从单体 → Monorepo → 微前端，平滑升级

### 2.4 应用启动流程

```
main.ts bootstrap()
  │
  ├─ setupLoading()              # 0. 首屏加载动画
  ├─ setupMicroApp()             # 1. micro-app.start() ← 必须在 createApp 前
  ├─ createApp(App)              # 2. 创建 Vue 实例
  ├─ setupStore(app)             # 3. Pinia + 持久化
  ├─ setupRequestCore(app)       # 4. Axios 请求核心
  ├─ setupLayoutSystem(app)      # 5. 布局系统
  ├─ setupNaiveUI(app)           # 6. Naive UI 全局通知
  ├─ setupDirectives(app)        # 7. 自定义指令
  ├─ router.isReady()            # 8. 等待路由就绪
  └─ app.mount('#app')           # 9. 挂载
```

> **关键**：`setupMicroApp()` 必须在 `createApp()` 之前调用，确保 `<micro-app>` 自定义元素在 Vue 编译前注册。

```
my-platform/
├── main-app/                     # 主应用（统一入口、鉴权、路由）
│   ├── src/
│   │   ├── config/               # 子应用注册表（地址/环境/元信息）
│   │   ├── plugins/              # micro-app.start() 插件

│   │   └── types/                # Window 扩展 + 通信类型声明

│

│   ├── sub-app-a/                # 子应用 A（任意技术栈）
│   ├── sub-app-b/                # 子应用 B
│   └── sub-app-c/                # 子应用 C
│
├── shared/                       # 主子应用共享包（关键！）
│   ├── types/                    # 通信数据接口类型定义
│   ├── constants/                # PostMessage 消息类型常量枚举
│   └── utils/                    # 共享工具函数（格式化/鉴权工具等）
│
└── docs/                         # 架构文档
    ├── architecture.md           # 架构总览

    └── onboarding.md             # 子应用接入规范
```

**官方核心建议**：

- `shared/` 包是**微前端协作的关键契约层**，主应用团队维护，子应用团队消费
- 子应用应尽量保持**技术无关性**，不依赖主应用的内部模块
- 通信协议（消息类型 + payload 格式）应文档化并版本化
- 子应用配置表（注册表）建议提取到独立文件，方便未来改为**后端动态配置**

### 3.4 `shared/` 契约层详解

`shared/` 是微前端协作的**关键契约层**，主应用团队维护，所有子应用消费。

#### 路径别名配置

```typescript
// 主应用 tsconfig/tsconfig.app.json
"paths": { "@shared/*": ["shared/*"] }

// 子应用 sys-mock/logistics/tsconfig.json
"paths": { "@shared/*": ["../../shared/*"] }
```

#### 常量使用示例

```typescript
// shared/constants/index.ts 定义
export const MESSAGE_TYPES = {
  MICRO_APP_NAVIGATE: 'MICRO_APP_NAVIGATE',  // 子→主：路由跳转
  CUSTOM_MESSAGE: 'CUSTOM_MESSAGE',          // 子→主：自定义消息
  DATA_UPDATE: 'DATA_UPDATE',                // 子→主：数据推送
  MOUNTED: 'MOUNTED',                        // 子→主：挂载完成
  THEME_CHANGE: 'THEME_CHANGE',              // 主→子：主题变更
  CUSTOM_MESSAGE_ACK: 'CUSTOM_MESSAGE_ACK',  // 主→子：确认回执
  // ...
} as const

// 主应用容器中使用
import { MESSAGE_TYPES } from '@shared/constants'
case MESSAGE_TYPES.MICRO_APP_NAVIGATE:
  handleNavigate(payload.path)
  break

// 子应用中使用
import { MESSAGE_TYPES } from '@shared/constants'
sendMessageToMainApp({ type: MESSAGE_TYPES.CUSTOM_MESSAGE, payload: { ... } })
```

#### 类型使用示例

```typescript
// shared/types/index.ts 定义
export interface MicroAppData {
  token: string
  userInfo: Record<string, unknown>
  theme: ThemeData
  env: EnvData
  appInfo: AppInfo
  // ...
}

// 主应用中类型约束
import type { MicroAppData } from '@shared/types'
const appData = computed<MicroAppData>(() => ({ ... }))
```

### 4.2 微前端启动插件 — `micro-app.ts`

```typescript
import microApp from '@micro-zoe/micro-app'

export function setupMicroApp() {
  microApp.start({
    'disable-memory-router': false, // 保留虚拟路由系统
    'disable-patch-request': false, // 保留请求补全（资源路径修正）
  })
}
```

**官方可用配置项**（按需扩展）：

| 配置项                  | 类型    | 默认值 | 说明                         |
| ----------------------- | ------- | ------ | ---------------------------- |
| `iframe`                | boolean | false  | 全局开启 iframe 沙箱         |
| `inline`                | boolean | false  | 使用 inline script（调试用） |
| `destroy`               | boolean | false  | 卸载时销毁缓存               |
| `disable-scopecss`      | boolean | false  | 关闭样式隔离                 |
| `disable-sandbox`       | boolean | false  | 关闭沙箱（不推荐）           |
| `disable-memory-router` | boolean | false  | 关闭虚拟路由                 |
| `disable-patch-request` | boolean | false  | 关闭请求补全                 |
| `fiber`                 | boolean | false  | 开启异步渲染（长任务分片）   |
| `ssr`                   | boolean | false  | 服务端渲染模式               |
| `global-assets`         | object  | —      | 全局共享的 JS/CSS 资源       |

### 4.3 微应用容器 — `micro-app/index.vue`

核心功能：

- 从路由参数 `:appId` 获取子应用 ID
- 通过 `getMicroAppUrl()` 解析子应用地址
- 通过 `appData` computed 向子应用注入数据（token/主题/工具函数）
- PostMessage 监听子应用事件（6 种消息类型）
- `watch(themeStore.mode)` 主题变化实时推送

```vue
<micro-app
  v-if="appUrl"
  :name="appId"
  :url="appUrl"
  :data="appData"
  iframe
  keep-alive
  @mounted="handleMounted"
  @unmount="handleUnmount"
  @error="handleError"
  @datachange="handleDataChange"
/>
```

**关键属性说明**：

| 属性          | 说明                                       |
| ------------- | ------------------------------------------ |
| `name`        | 子应用唯一标识，同一个 name 对应同一个实例 |
| `url`         | 子应用入口地址（HTML）                     |
| `data`        | 主 → 子数据传递（响应式，变更自动推送）    |
| `iframe`      | 启用 iframe 沙箱（Vite 推荐）              |
| `keep-alive`  | 保活模式，切换时保留子应用状态             |
| `@mounted`    | 子应用挂载完成事件                         |
| `@unmount`    | 子应用卸载事件                             |
| `@error`      | 子应用加载失败事件                         |
| `@datachange` | 子应用通过 `dispatch` 推送数据变更         |

**官方更多可用属性**：

| 属性           | 说明                                                    |
| -------------- | ------------------------------------------------------- |
| `default-page` | 子应用默认页面路径                                      |
| `baseroute`    | 子应用路由前缀                                          |
| `router-mode`  | 路由模式：`search` / `native` / `native-scope` / `pure` |
| `clear-data`   | 卸载时清空通信数据                                      |
| `fiber`        | 单应用级别开启异步渲染                                  |
| `inline`       | 单应用级别开启内联 script                               |
| `destroy`      | 卸载时彻底销毁                                          |

## 五、主子应用通信

### 5.1 通信架构全景

```
                   ┌──────────────────────────────┐
                   │         micro-app 框架层       │
                   │   (代理 data / dispatch / 事件) │
                   └────────────┬─────────────────┘
                                │
         ┌──────────────────────┼──────────────────────┐
         │                      │                      │
    ① data 属性绑定         ② setData/getData       ③ PostMessage
    (主 → 子, 响应式)       (命令式, 双向)          (子 → 主, 跨 iframe)
         │                      │                      │
         ▼                      ▼                      ▼
    appData computed      watch 主题同步          6 种消息类型
    (token/theme/env)     microApp.setData()     (NAVIGATE/DATA/...)
```

### 5.2 通信方式一：data 属性绑定（主 → 子）

**推荐用于**：初始化数据、用户信息、主题状态等需要响应式更新的场景。

```typescript
// 主应用 micro-app/index.vue
const appData = computed(() => ({
  token: userStore.token,
  userInfo: userStore.userInfo,
  theme: { mode: themeStore.mode, isDark: themeStore.isDark },
  env: { mode: import.meta.env.MODE, baseUrl, apiUrl },
  appInfo: { mainApp: 'Robot Admin', version: '1.11.0' },
  utils: { formatTime },
  methods: { showMessage, navigateToMain },
}))
```

```typescript
// 子应用接收
if (window.__MICRO_APP_ENVIRONMENT__) {
  window.microApp?.addDataListener(data => {
    appStore.setMainAppData(data)
    // 主题同步
    if (data.theme?.isDark !== undefined) {
      document.documentElement.classList.toggle('dark', data.theme.isDark)
    }
  })
}
```

> **官方说明**：data 属性只传递变更的数据，micro-app 内部会做**浅合并**（Object.assign）。

### 5.3 通信方式二：setData / getData（命令式）

**推荐用于**：非渲染触发的数据同步，如主题切换、手动推送。

```typescript
// 主应用主动推送
microApp.setData('logistics', { theme: { isDark: true } })

// 主应用获取子应用数据
const childData = microApp.getData('logistics')
```

```typescript
// 子应用主动推送
window.microApp?.dispatch({ type: 'DATA_UPDATE', payload: { ... } })

// 子应用获取主应用数据
const mainData = window.microApp?.getData()
```

> **注意**：`setData` 会与上一次数据做浅合并，并触发子应用的 `addDataListener`。使用 `forceDispatch` 可在子应用未渲染时强制发送。

### 5.4 通信方式三：PostMessage（跨 iframe）

**推荐用于**：iframe 沙箱下的事件通知，如路由跳转、自定义业务消息。

当前已实现 6 种消息类型：

| 消息类型             | 方向    | 用途                              |
| -------------------- | ------- | --------------------------------- |
| `MICRO_APP_NAVIGATE` | 子 → 主 | 子应用请求主应用路由跳转          |
| `CUSTOM_MESSAGE`     | 子 → 主 | 自定义业务消息                    |
| `DATA_UPDATE`        | 子 → 主 | 子应用推送数据到门户页            |
| `MOUNTED`            | 子 → 主 | 子应用挂载完成通知                |
| `ROUTE_CHANGE`       | 子 → 主 | 子应用路由变化通知                |
| 主题同步             | 主 → 子 | `microApp.setData()` 推送暗色模式 |

```typescript
// 子应用发送 PostMessage
window.parent?.postMessage(
  {
    type: 'MICRO_APP_NAVIGATE',
    payload: { path: '/portal' },
  },
  '*'
)
```

```typescript
// 主应用监听 PostMessage
window.addEventListener('message', event => {
  const { type, payload } = event.data || {}
  switch (type) {
    case 'MICRO_APP_NAVIGATE':
      router.push(payload.path)
      break
    case 'CUSTOM_MESSAGE':
      handleCustomMessage(payload, event.source)
      break
    case 'DATA_UPDATE':
      handleDataUpdate(payload, event.source)
      break
  }
})
```

### 5.5 通信方式四：全局数据（待实现）

官方提供的全局数据机制，适用于**多个子应用间**共享数据：

```typescript
// 主应用设置全局数据
microApp.setGlobalData({ user: userInfo, locale: 'zh-CN' })

// 任意子应用获取
const globalData = window.microApp?.getGlobalData()

// 任意子应用监听变化
window.microApp?.addGlobalDataListener(data => {
  console.log('全局数据变更:', data)
})
```

> 当前项目**尚未实现**全局数据，建议在 Phase 2 接入。

### 5.6 通信最佳实践

| 场景                          | 推荐方式        | 原因                      |
| ----------------------------- | --------------- | ------------------------- |
| 初始化注入（token/主题/环境） | `data` 属性绑定 | 响应式，自动同步          |
| 主题切换实时同步              | `setData`       | 精准推送，不重新渲染      |
| 子应用请求路由跳转            | PostMessage     | iframe 沙箱隔离了 history |
| 子应用推送业务数据            | PostMessage     | 异步，不阻塞渲染          |
| 多子应用间共享数据            | globalData      | 广播机制，一对多          |
| 子应用挂载完成通知            | `@mounted` 事件 | 框架级生命周期回调        |

## 六、沙箱与样式隔离

### 6.1 沙箱模式选择

micro-app 提供两种沙箱模式：

| 模式                  | 原理                       | 优势     | 劣势           | 适用场景              |
| --------------------- | -------------------------- | -------- | -------------- | --------------------- |
| **with 沙箱**（默认） | Proxy 代理 window          | 性能较高 | `var` 声明泄漏 | 非 Vite 项目          |
| **iframe 沙箱**       | 创建 iframe 隔离 JS 上下文 | 完全隔离 | 性能略低 ~50ms | **Vite 项目（推荐）** |

> **Robot Admin 使用 iframe 沙箱**，因为 Vite 8 的 ESM 模块在 with 沙箱下存在兼容问题。

### 6.2 样式隔离策略

iframe 沙箱天然隔离 CSS，无需额外配置。主子应用间的样式互通通过 **CSS 变量** 实现：

```scss
// 主应用 styles/theme-variables.scss
:root {
  --portal-primary: var(--primary-color, #2080f0);
  --portal-success: var(--success-color, #18a058);
  --portal-bg-body: var(--body-color, #ffffff);
  --portal-text-1: var(--text-color-1, #262626);
}

html.dark {
  --portal-bg-body: #18181c;
  --portal-text-1: #e8e8e8;
}
```

```scss
// 子应用消费 CSS 变量（通过 data 注入或独立定义）
.logistics-header {
  background: var(--portal-bg-body, #ffffff);
  color: var(--portal-text-1, #262626);
}
```

### 6.3 Dark Mode 同步

```typescript
// 主应用 watch 监听
watch(
  () => themeStore.mode,
  newMode => {
    if (window.microApp) {
      microApp.setData(appId.value, {
        ...appData.value,
        theme: { mode: newMode, isDark: newMode === 'dark' },
      })
    }
  }
)
```

```typescript

  if (data.theme?.isDark !== undefined) {
    document.documentElement.classList.toggle('dark', data.theme.isDark)
  }
})
```

## 七、性能优化策略

### 7.1 预加载（preFetch）

官方提供 `preFetch` API，可在空闲时预加载子应用资源，**首次切换提速 60%+**：

```typescript
import microApp from '@micro-zoe/micro-app'

// 在主应用启动后预加载
microApp.preFetch([
  { name: 'logistics', url: 'http://localhost:3003' },
  { name: 'warehouse', url: 'http://localhost:3004' },
])
```

> **当前状态**：⬜ 未实现。建议在 `setupMicroApp()` 中根据 `getAllMicroApps()` 自动预加载。

### 7.2 Keep-Alive 保活

已通过 `<micro-app keep-alive>` 实现，子应用切换时不会销毁，再次打开瞬间恢复。

**注意事项**：

- keep-alive 会增加内存占用，高频使用的 3-5 个应用适合保活
- 低频应用建议设置 `destroy` 属性，卸载时释放资源
- 可通过 `clear-data` 属性在卸载时清空通信数据

### 7.3 Fiber 异步渲染

官方提供 `fiber` 模式，将 JS 执行拆分为多个微任务，**避免长任务阻塞主线程**：

```typescript
// 全局开启
microApp.start({ fiber: true })

// 或单应用开启
<micro-app name="heavy-app" url="..." fiber />
```

> **适用场景**：子应用 JS 体积大（>1MB）、首次渲染耗时长的场景。

### 7.4 全局资源共享（globalAssets）

将 Vue、Naive UI 等公共依赖配置为全局资源，**所有子应用共享，减少重复加载**：

```typescript
microApp.start({
  globalAssets: {
    js: ['https://cdn.example.com/vue@3.5.13/vue.global.prod.js'],
    css: ['https://cdn.example.com/common-styles.css'],
  },
})
```

> **适用场景**：多个子应用使用相同版本的 Vue/Naive UI 时，通过 CDN 共享可减少 30%+ 总包体积。

### 7.5 性能优化清单

| 优化项                       | 预期收益          | 优先级 | 状态      |
| ---------------------------- | ----------------- | ------ | --------- |
| 子应用预加载（preFetch）     | 首次切换提速 60%  | P0     | ⬜ 待实现 |
| Keep-Alive 保活              | 切换零延迟        | P0     | ✅ 已实现 |
| Fiber 异步渲染               | 避免长任务阻塞    | P1     | ⬜ 待实现 |

### 12.2 建议封装的模块

#### 1. `useMicroAppPreFetch` — 预加载 Hook

```typescript
// composables/useMicroAppPreFetch.ts
export function useMicroAppPreFetch() {
  const apps = getAllMicroApps()
  const preFetchList = apps.map(app => ({
    name: app.id,
    url: getMicroAppUrl(app.id) || app.dev,
  }))

  onMounted(() => {
    // 空闲时预加载
    requestIdleCallback(() => {
      microApp.preFetch(preFetchList)
    })
  })
}
```

#### 3. 增强错误边界

```typescript
  // 处理消息...
})
```

## 十三、常见问题与排错

### Q1：子应用静态资源 404

**原因**：子应用的 CSS/图片使用了相对路径，被主应用的 URL 解析拦截。

**解决**：子应用 `vite.config.ts` 设置 `base` 为子应用完整 URL。

```typescript
export default defineConfig({
  base:
    process.env.NODE_ENV === 'production'
      ? 'https://logistics.example.com/'
      : '/',
})
```

### Q2：子应用 HMR 热更新不生效

**原因**：micro-app 沙箱拦截了 WebSocket 连接。

**解决**：确保 `disable-patch-request: false`（已默认配置）。

### Q3：ESLint 报 `<micro-app>` 组件名错误

**原因**：vue/component-name-in-template-casing 要求 PascalCase。

**解决**：在 `<micro-app>` 上方添加：

```html
<!-- eslint-disable-next-line vue/component-name-in-template-casing -->
<micro-app ... />
```

### Q4：子应用全局变量污染主应用

**原因**：with 沙箱对 `var` 声明的全局变量无法拦截。

**解决**：使用 `iframe` 沙箱模式（已配置为默认）。

### Q5：子应用路由跳转不生效

**原因**：iframe 沙箱中 `history` 是隔离的。

**解决**：通过 PostMessage 通知主应用跳转。

```typescript
window.parent?.postMessage(
  {
    type: 'MICRO_APP_NAVIGATE',
    payload: { path: '/portal' },
  },
  '*'
)
```

### Q6：子应用 Vue Devtools 无法调试

**原因**：iframe 沙箱隔离了 devtools 注入。

**解决**：

1. 子应用独立访问 `http://localhost:3003` 调试
2. 或使用 `inline` 模式临时替换 `iframe` 模式（仅开发环境）

### Q7：微应用之间样式冲突

**原因**：多个子应用同时渲染时，全局样式互相影响。

**解决**：iframe 沙箱天然隔离，不存在此问题。如使用 with 沙箱，需启用 `disable-scopecss: false`。

### Q8：子应用加载白屏

**排查步骤**：

1. 检查子应用是否独立可访问（直接访问子应用 URL）
2. 检查 CORS 响应头是否正确
3. 检查主应用控制台是否有跨域错误
4. 检查 `getMicroAppUrl()` 返回值是否正确
5. 检查网络面板中子应用资源是否 200
